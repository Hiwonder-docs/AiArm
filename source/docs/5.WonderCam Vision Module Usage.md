# 5. WonderCam Vision Module Usage

## 5.1 Color Recognition

1. In the preceding lessons, we covered the hardware details and functional features of the CoreX main controller and WonderCam vision recognition module. In this lesson, we will integrate the main controller with the vision module to delve into more AI applications.

2. After recognizing red, green and blue objects, WonderCam vision module will relay this information to the main controller whose RGB lights illuminate in corresponding colors.

### 5.1.1 Preparation

1)  Command WonderCam module to learn red, green and blue colors referring to “**1. WonderCam Vision Module Usage**”.

2)  Connect WonderCam vision module to port 9 of CoreX controller.

3)  Install WonderCode programming software according to the instructions provided in “**3. Scratch Programming**”.

### 5.1.2 **Program Logic**

1. The initial step involves initializing the vision module. This includes configuring the module's functionality, setting the initial position, and disabling the ultrasonic module. It is important to disable the ultrasonic module because its emission of light can potentially interfere with color recognition.

2. Following the initialization, the program proceeds to verify if the learned colors are being recognized. Based on the outcome of this verification, the hardware on the main controller provides corresponding feedback effects.

3. For detailed implementation of this game, please refer to the **5. Program Analysis** section.

3. **Recognition Feedback**

​	(1) To configure the recognition feedback, let**s consider the example of using light 0 in WonderCam**s barcode recognition process. When WonderCam detects a barcode, light 0 will be turned on for 0.5 seconds before turning off. At the same time, a variable called **LED light** will be incremented from 0 to 1, indicating the change from **LED=0** to **LED=1**.

​	(3) If the barcode is removed from WonderCam's recognition zone, indicating that no barcode is currently being recognized, the program will exit the loop. In this case, the variable **LED light** will be set to 0. WonderCam will then resume its barcode recognition process and perform the above operations again.
