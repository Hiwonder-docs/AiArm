# 3. Scratch Programming

## 3.1 CoreX Controller Introduction

Robots detect the surrounding environment and gather information through sensor, and then respond accordingly based on the external environment.

Based on this, Hiwonder has developed a CoreX controller, which is used to control these sensors.

1.  **Introduce to Electronic Module on Controller**

​	(1) The CoreX controller allows you to control the electronic modules and external modules through programming.

2.  **Controller Structure**

<img src="../_static/media/chapter_3/section_1/media/image2.png" class="common_img" style="width:800px"/>

​	(2) The controller consists of ESP32 microcontroller, buzzer, infrared emitter and receiver, sound module, two programmable buttons as well as six RGB light modules.

3.  **On-board Modules Instruction**

<img src="../_static/media/chapter_3/section_1/media/image3.jpeg" class="common_img" style="width:800px"/>

(1) **ESP32 microcontroller**

​	EPS32 microcontroller integrates WIFI functionality, supports Bluetooth, and features 32-bit dual-core processor, EPS32 microcontroller integrates WIFI functionality, supports Bluetooth, and features 32-bit dual-core processor, which is developed for more complicated application.

(2) **Buzzer**

​	A buzzer is an electronic device that produces sound. It can play music through program setting.

(3) **RGB light module**

​	An RGB light module is a compact electronic component that integrates red, green, and blue LEDs into a single module. The color output can be customized by program.

(4) **Infrared emitter and receiver**

​	An infrared emitter and receiver are electronic components used to transmit and receive infrared (IR) signals, and perform the desired action based on the received data. The commonly is widely used in applications such as TV remote controls and wireless data transmission between devices.

(5) **Sound sensor**

​	A sound sensor is an electronic component that detects and measures sound levels.

(6) **Programmable button**

​	CoreX controller contains two programmable buttons A and B that can be configured to perform different actions or functions based on user-defined settings or programming.

### 3.5.2 Introduction to the loop structure

1. The loop structure refers to a kind of program structure set for repeatedly executing a block of code multiple times. A loop statement is usually composed of a loop body and a termination condition, and whether it can continue repeating is determined by the termination condition of the loop.

2. It is generally divided into two types: judge the condition before repeat
3. Loop structures can be further divided into two types: pre-test loops and post-test loops. Pre-test loops first evaluate the condition before entering the loop, while post-test loops first perform the loop body and then evaluate the condition. They are particularly useful when dealing with iterative tasks or processing collections of data, enabling the program to leverage the computational strengths of computers.
